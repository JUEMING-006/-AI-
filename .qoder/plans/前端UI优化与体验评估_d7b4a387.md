# 前端 UI 优化与体验评估方案

## 设计方向（frontend-design 思维）

**受众**：高职大专学生（18-22岁），对手机交互有高期待，审美偏好活力但不花哨
**产品单一使命**：陪伴升本全程，消除焦虑、建立信心
**设计签名（Signature）**：「成长弧线」— 贯穿全应用的弧形渐变 + 进度感视觉语言

### 色彩系统升级

| 角色 | 旧值 | 新值 | 设计意图 |
|------|------|------|----------|
| Primary | #4E6EF2 | #5B7FFF | 更温暖的蓝紫，减少冰冷感 |
| Primary Gradient End | 无 | #8B5CF6 | 蓝→紫渐变体现成长进阶 |
| Background | #F5F6FA | #F8F9FE | 更柔和的微紫灰底 |
| Card | #FFFFFF | #FFFFFF 带 0.6 opacity blur | 毛玻璃质感呼应鸿蒙6.0液态玻璃 |
| Success | #34C759 | #10B981 | 更沉稳的翡翠绿 |
| Warning | #FF9500 | #F59E0B | 琥珀色更柔和 |
| Accent | 无 | #EC4899 | 点缀色用于重要数据高亮 |

### 动画策略（鸿蒙 API 23 可用能力）

| 场景 | 技术 | 效果 |
|------|------|------|
| 页面进入 | `.transition({ type: TransitionType.Insert, opacity: 0, translate: { y: 20 } })` | 子内容从下方淡入 |
| 卡片点击 | `animateTo({ duration: 200, curve: Curve.FastOutSlowIn })` + scale | 按下缩小反馈 |
| 倒计时数字 | `.animation({ duration: 600, curve: Curve.EaseOut })` | 数字变化时弹性过渡 |
| 列表加载 | 逐项 stagger 动画（延迟递增） | 列表项依次滑入 |
| Tab 切换 | `.tabBar` 图标 `SymbolEffect(BounceSymbolEffect)` | 切换时图标弹跳 |
| 进度更新 | `animateTo` 驱动 Progress value | 进度条平滑增长 |
| 毛玻璃 | `.backgroundBlurStyle(BlurStyle.COMPONENT_REGULAR)` | 顶部导航栏/卡片背景 |

### 排版体系

| 角色 | 字号 | 字重 | 用途 |
|------|------|------|------|
| Display | 28fp | Bold | 倒计时天数、核心数据 |
| Title | 20fp | Medium | 区块标题 |
| Subtitle | 16fp | Medium | 卡片标题 |
| Body | 14fp | Normal | 正文内容 |
| Caption | 12fp | Normal | 辅助信息、标签 |

## 实施任务分解

### Task 1: 设计系统升级 + 动画工具层

**范围**：更新 Constants.ets 色彩系统，创建 `common/AnimationHelper.ets` 动画工具，创建 `common/DesignTokens.ets` 排版/间距令牌

- 更新 `AppColors` 为新色彩方案，添加渐变色对
- 创建 `AnimationHelper.ets`：封装 stagger 动画、卡片按压效果、淡入滑入过渡的配置常量
- 创建 `DesignTokens.ets`：字体大小/字重/圆角/间距/阴影的统一令牌
- 更新 `CommonCard.ets`：添加 backgroundBlurStyle 毛玻璃 + 按压缩放动画

### Task 2: 首页视觉升级

**范围**：HomePage.ets 全面优化

- 倒计时卡片：大弧形渐变背景（蓝→紫），数字使用 Display 字号 + 弹性动画
- 快捷入口 Grid：图标添加渐变圆形底色 + 点击涟漪缩放
- 政策卡片：左侧彩色竖条 + 毛玻璃背景 + 进入动画
- 升本价值 Swiper：卡片使用大圆角 + 背景模糊叠加

### Task 3: 院校库模块视觉升级

**范围**：CollegePage/CollegeDetailPage/CollegeComparePage

- 院校卡片：添加 hover/press 缩放动画，收藏按钮心跳效果
- 筛选标签：选中态渐变背景 + 切换过渡动画
- 详情页：分数线数据区域使用柱状可视化 + 数字动态计数
- 对比页：横向滑动表格 + 差异高亮

### Task 4: 学习规划 + 志愿匹配视觉升级

**范围**：StudyPlanPage/DailyTaskPage/ProgressPage/VolunteerPage/ScoreInputPage/VolunteerResultPage

- 三阶段进度：弧形进度环（替代线性 Progress）+ 阶段切换动画
- 每日任务：Checkbox 完成时 ✓ 弹跳 + 划线动画
- 柱状图：柱子从底部生长动画（stagger）
- 志愿三梯度：彩色渐变分组头 + 概率环动画 + 卡片入场

### Task 5: 创建高职学生用户智能体 + UX 评估

**范围**：创建 beginner-learner-simulator 自定义智能体模拟高职学生视角，对优化后的软件进行体验评估

- 用户画像：20岁高职大三学生，每天重度使用手机，对卡顿/丑UI零容忍，期望"好用又好看"
- 评估维度：首次打开印象、导航直觉性、信息层次清晰度、动画是否过度、功能可发现性
- 输出 UX 评估报告 + 改进建议

## 依赖关系

```
Task 1 (设计系统) → Task 2/3/4 (可并行) → Task 5 (UX 评估)
```

## 需要联网查找的鸿蒙动画 API

在实施中需从 developer.huawei.com 确认以下 API 在 API 23 的具体用法：
- `transition()` 组件转场动画详细参数
- `SymbolGlyph` 的 `symbolEffect` 动画类型
- `Progress` 组件自定义样式（环形/弧形）
- `Column.sharedTransition()` 共享元素过渡（列表→详情）
