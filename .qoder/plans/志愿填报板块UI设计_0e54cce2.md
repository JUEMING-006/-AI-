# 志愿填报板块 前端UI设计方案

## 一、设计背景与定位

**产品定位**：专升本智能志愿填报助手（区别于高考志愿系统，专升本特点：省内院校为主、专业对口限制、招生人数少、竞争集中）

**参考竞品**：夸克高考志愿（冲稳保三梯度、院校优先/专业优先双模式）、志愿星图（多维筛选、志愿单分析）

**设计原则**：延续现有液态玻璃风格 + 三梯度配色体系，保持与项目整体UI一致

---

## 二、页面架构总览（新增5个页面）

```
views/volunteer/
├── VolunteerPage.ets          [改造] 志愿主页 → 功能入口Hub
├── ScoreInputPage.ets         [保留] 成绩录入

├── MajorDetailPage.ets        [新增] 专业详情页
└── CollegeVolunteerPage.ets   [新增] 院校志愿详情页
```

## 三、各页面UI设计详案

### 3.1 VolunteerPage（志愿主页 - 功能Hub改造）

**设计理念**：从简单的"录入成绩+查看推荐"升级为完整的志愿填报工作台

**页面结构**：
```
Scroll
└── Column
    ├── [顶部Banner] 个人成绩摘要卡片
    │   ├── 左侧：总分 + 省份 + 排名区间
    │   ├── 右侧：录取概率环形图（综合评估）
    │   └── 底部操作栏："修改成绩" / "首次录入" 按钮
    │
    ├── [核心功能区] 四宫格快捷入口
    │   ├── 智能选志愿（图标 + 副标题"冲稳保方案"）
    │   ├── 我的志愿表（图标 + 副标题"已选X所"）
    │   ├── 分数线查询（图标 + 副标题"历年数据"）
    │   └── 志愿诊断（图标 + 副标题"风险评估"）
    │
    ├── [推荐概览] 冲稳保方案预览卡片
    │   ├── 三色横向进度条（冲:稳:保 比例可视化）
    │   ├── 每个梯度：院校数量 + 代表院校名
    │   └── "查看完整方案" 按钮
    │
    ├── [热门院校] 水平滚动院校卡片列表
    │   ├── 院校缩略卡片（校名 + 去年分数线 + 招生人数）
    │   └── 每张卡片可点击进入院校志愿详情
    │
    └── [政策提醒] 填报时间线卡片
        ├── 倒计时：距离志愿填报截止还有X天
        └── 关键节点时间提示（报名、填报、录取）
```

**交互细节**：
- 四宫格入口使用 CommonCard + Grid(2x2) 布局，每个格子带液态玻璃效果
- 成绩摘要卡片使用渐变背景（PRIMARY_COLOR → PRIMARY_GRADIENT_END）
- 未录入成绩时，顶部卡片显示"快速录入成绩，开启智能推荐"引导文案

### 3.2 VolunteerResultPage（智能选志愿 - 全面升级）

**设计理念**：参考夸克"院校优先/专业优先"双模式，增加筛选维度

**页面结构**：
```
Column
├── [顶部] 模式切换 Tab
│   ├── Tab1: 院校优先（按录取概率排序）
│   └── Tab2: 专业优先（按专业匹配度排序）
│
├── [筛选栏] 横向滚动筛选标签
│   ├── 院校类型：公办 / 民办 / 全部
│   ├── 地区：省内城市多选
│   ├── 学费区间：<1万 / 1-2万 / >2万
│   └── 招生人数：>50 / >100 / 不限
│
├── [统计栏] 冲稳保数量统计横条
│   ├── 冲刺 X 所 | 稳妥 X 所 | 保底 X 所
│   └── 颜色编码与现有一致（红/绿/蓝）
│
├── [列表区] LazyForEach 院校卡片列表
│   ├── 按梯度分组（带分组标题"冲刺院校"/"稳妥院校"/"保底院校"）
│   ├── 每张院校卡片：
│   │   ├── 左侧Column：院校名 + 专业名 + 标签（公办/民办）
│   │   ├── 右侧Column：录取概率百分比 + 去年最低分
│   │   ├── 底部Row：招生人数 + 学费 + "加入志愿表"按钮
│   │   └── 右上角：收藏/对比 图标
│   └── Stagger入场动画保持现有实现
│
└── [底部悬浮] 操作栏
    ├── 左侧：已选 X/Y 所（限制最多5个志愿）
    └── 右侧："生成志愿表" 按钮（渐变色）
```

**交互细节**：
- 筛选标签使用横向 Scroll + 胶囊按钮样式
- 切换"院校优先/专业优先"时有平滑过渡动画
- 卡片点击可展开显示更多信息（近三年分数线折线图预留区域）
- "加入志愿表"按钮按下时有弹性收缩动画 + 底部数量+1反馈

### 3.3 VolunteerFormPage（我的志愿表 - 新增）

**设计理念**：参考夸克"志愿表"，可管理、排序、导出已选院校

**页面结构**：
```
Column
├── [顶部] 志愿表标题 + 操作按钮
│   ├── "我的志愿表（X/5）"
│   └── 右侧："诊断" + "清空" 按钮
│
├── [排序提示] 横幅
│   └── "拖动调整志愿顺序，第1志愿优先录取"
│
├── [志愿列表] 可拖拽排序列表
│   ├── 每个志愿项：
│   │   ├── 左侧：序号圆圈（1-5）+ 梯度色标
│   │   ├── 中间Column：
│   │   │   ├── 院校名称（粗体）
│   │   │   ├── 专业名称
│   │   │   └── Row：去年分数线 + 录取概率标签
│   │   ├── 右侧：拖拽手柄图标 + 删除按钮
│   │   └── 底部：梯度标签（冲刺/稳妥/保底 带颜色）
│   └── 空状态：EmptyView "还没有添加志愿，去智能选志愿吧"
│
├── [风险分析] 梯度分布卡片
│   ├── 横向进度条显示冲:稳:保比例
│   ├── 评分（如"A-方案较为稳健"）
│   └── 建议文案："建议增加1所保底院校"
│
└── [底部] 操作按钮组
    ├── "志愿诊断" 次要按钮
    └── "确认提交" 主按钮（渐变色）
```

**交互细节**：
- 列表项支持长按拖拽排序（使用 List + onItemDragStart）
- 左滑显示删除按钮
- 梯度分布实时更新（每次增删/排序后重新计算）
- 空志愿表时底部按钮置灰不可点击

### 3.4 ScoreLinePage（分数线查询 - 新增）

**设计理念**：参考夸克"查大学"的分数线展示，适配专升本场景

**页面结构**：
```
Column
├── [搜索栏] SearchBar 组件
│   └── placeholder: "搜索院校名称"
│
├── [筛选条件] Row
│   ├── 年份选择：Picker（2024/2023/2022）
│   ├── 院校类型：Segment（公办/民办/全部）
│   └── 专业类别：下拉选择
│
├── [我的分数参考线] 卡片（仅录入成绩后显示）
│   ├── "我的总分：XXX分"
│   └── 虚线标注在列表中的位置参考
│
├── [院校分数线列表] List
│   ├── 每项结构：
│   │   ├── Row：院校名 + 标签（公办/民办）
│   │   ├── Row：
│   │   │   ├── 2024年最低分 | 2023年最低分 | 2022年最低分
│   │   │   └── 分数趋势箭头（↑升/↓降/→平）
│   │   ├── Row：招生计划人数 + 报录比
│   │   └── 与我的分差："+15分" 或 "-8分"（颜色区分安全/危险）
│   └── 按分数线从高到低排序
│
└── [底部说明] 数据来源与免责声明文字
```

**交互细节**：
- 搜索支持实时过滤（debounce 300ms）
- 分数趋势用微型折线图或箭头图标表示
- 与我的分差：正数(绿色安全)、负数(红色风险)、0-10分(橙色需冲刺)
- 点击某院校行可展开详细专业分数线

### 3.5 VolunteerDiagnosePage（志愿诊断 - 新增）

**设计理念**：参考夸克"志愿诊断"，帮助评估方案风险

**页面结构**：
```
Column
├── [综合评分] 大卡片
│   ├── 中心：综合评分（如 85分 / A级）
│   ├── 环形进度条包裹分数
│   └── 评语："方案整体稳健，建议微调"
│
├── [维度分析] 多维度卡片列表
│   ├── 梯度合理性（冲稳保比例是否合理）
│   │   ├── 当前比例可视化
│   │   └── 建议比例对比
│   │
│   ├── 分数匹配度（各志愿与分数的匹配情况）
│   │   ├── 每个志愿的分差柱状图
│   │   └── 标注风险区间
│   │
│   ├── 志愿梯度（相邻志愿分差是否合理）
│   │   ├── 阶梯图展示志愿分数递减趋势
│   │   └── 标注梯度过大/过小的位置
│   │
│   └── 滑档风险（综合评估滑档概率）
│       ├── 风险等级标签（低/中/高）
│       └── 风险点说明
│
├── [具体建议] 优化建议列表
│   ├── 每条建议：图标 + 建议文案 + 操作按钮
│   │   例："第2志愿与第3志愿分差仅2分，建议拉开梯度"
│   │       → [去调整] 按钮跳转志愿表
│   └── 建议按优先级排序（红色紧急/橙色建议/绿色确认）
│
└── [底部] "返回调整志愿" 按钮
```

**交互细节**：
- 综合评分使用动画数字递增效果（0→85）
- 维度卡片支持点击展开/收起详情
- 建议项的操作按钮直接跳转到对应页面

## 六、实现优先级

| 优先级 | 页面 | 理由 |
|--------|------|------|
| P0 | VolunteerPage 改造 | 功能Hub是所有入口 |
| P0 | VolunteerResultPage 升级 | 核心功能"智能选志愿" |
| P1 | VolunteerFormPage | 志愿管理是完整流程闭环 |
| P1 | ScoreLinePage | 高频查询需求 |
| P2 | VolunteerDiagnosePage | 增值功能 |
| P2 | MajorDetailPage / CollegeVolunteerPage | 信息补充页 |

## 七、视觉风格要点

- 延续液态玻璃(GlassEffect)卡片风格
- 三梯度配色贯穿所有页面：冲刺(#FF6B6B) / 稳妥(#10B981) / 保底(#5B7FFF)
- 四宫格入口使用渐变背景 + 半透明图标
- 所有按钮保持按压反馈动画（AnimConfig配置）
- 列表使用 Stagger 入场动画
- 数据加载使用骨架屏占位
