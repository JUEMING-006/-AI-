# UI视觉风格优化方案

## 设计理念

融合三大先进审美方向：
- **Apple Vision Pro 液态玻璃**：半透明磨砂质感、柔和光影、真实物理深度
- **HarmonyOS 6 沉浸光感**：系统级视觉统一、原生玻璃材质、沉浸式体验
- **2026 AI应用设计趋势**：Glassmorphism 2.0深色基底+半透明面板、微动画反馈、内容优先的克制设计

核心原则：**统一、克制、有深度**。消除当前两套颜色体系并存的混乱，建立一致的视觉语言；通过层次感和微动画提升质感，而非堆砌视觉效果。

---

## Task 1: 配色系统统一与现代化

**目标**：消除Constants.ets两套颜色体系并存问题，所有硬编码颜色改为`$r()`资源引用，确保暗色模式自动适配。

**文件**：
- `entry/src/main/resources/base/element/color.json`
- `entry/src/main/resources/dark/element/color.json`
- `entry/src/main/ets/common/Constants.ets`

**具体改动**：

1. **扩展color.json**，添加缺失颜色定义：
   - `moon_top_bar_bg` 暗色值补充（当前亮色#FFFFFF，暗色需#1F1F1F）
   - `moon_function_tag_bg` 功能标签背景（亮色#FFFFFF，暗色#2A2A2A）
   - `moon_value_card_bg` 价值卡片背景（亮色#FFFFFF，暗色#1F1F1F）
   - `moon_icon_bg` 图标按钮背景（亮色#E8E8E8，暗色#2A2A2A）— 替代硬编码#E8E8E8
   - `moon_chat_bar_bg` 聊天顶栏背景（亮色#F8F8F8，暗色#1A1A1A）— 替代硬编码#F8F8F8
   - `moon_card_border` 卡片边框（亮色#E8E8E8，暗色#333333）
   - 统一功能色：`color_success` 改为#34C759（iOS绿），与ChatPage使用一致；或全项目统一为#4CAF50

2. **清理Constants.ets**：
   - 移除`AppColors`中未使用的旧Apple色系（TEXT_PRIMARY=#1D1D1F, TEXT_SECONDARY=#86868B, TEXT_HINT=#AEAEB2, GLASS_*系列）
   - 保留`PRIMARY_COLOR=#0066FF`等仍在使用的常量，但标注"已迁移至资源引用，仅用于兼容"
   - 统一功能色：`SUCCESS_COLOR`改为与color.json一致

3. **修正KimiGradient**（`DesignTokens.ets`）：
   - 当前START_COLOR和END_COLOR都是#0066FF，无渐变效果
   - 改为有意义的微妙渐变：START=#0066FF（Kimi蓝），END=#0052CC（深一档蓝）
   - 或保持纯色但移除linearGradient调用，避免误导

## Task 2: 液态玻璃质感升级（Glassmorphism 2.0）

**目标**：统一所有组件的液态玻璃参数，增强深度层次感和真实性。

**文件**：
- `entry/src/main/ets/common/DesignTokens.ets`
- `entry/src/main/ets/components/chat/ModelDropdownCard.ets`
- `entry/src/main/ets/components/chat/ChatTopBar.ets`
- `entry/src/main/ets/components/chat/ChatInputBar.ets`
- `entry/src/main/ets/components/chat/SideMenu.ets`
- `entry/src/main/ets/views/setting/SettingPage.ets`

**具体改动**：

1. **定义统一的液态玻璃参数规范**（DesignTokens.ets）：
```
// 主容器玻璃（输入栏、侧边栏入口、设置分组）
GLASS_CONTAINER = {
  blurStyle: COMPONENT_THICK,
  radius: 30,
  saturation: 2.0,
  brightness: 1.05,
  color: '#30FFFFFF'  // 暗色模式自动切换需通过资源引用
}

// 悬浮卡片玻璃（模型选择、下拉菜单）
GLASS_FLOATING = {
  blurStyle: COMPONENT_THICK,
  radius: 35,
  saturation: 2.2,
  brightness: 1.08,
  color: '#50FFFFFF'
}

// 按钮玻璃（顶栏按钮、图标按钮）
GLASS_BUTTON = {
  blurStyle: COMPONENT_THIN,
  radius: 20,
  saturation: 1.5,
  brightness: 1.05,
  color: '#20FFFFFF'
}
```

2. **统一ModelDropdownCard参数**：将`radius:25, saturation:1.8, brightness:1.0, color:#50FFFFFF`对齐为GLASS_FLOATING规范

3. **优化边框和高光**：
   - 当前问题：白色边框#66FFFFFF在白底上不可见
   - 改为：深色半透明边框`#66000000`（40%黑），1.5px宽度，确保在亮/暗模式下均可见
   - 添加顶部1px高光`#33FFFFFF`模拟玻璃边缘反光

4. **增强阴影深度**：统一阴影参数为3层叠加：
   - 内阴影：inset 0 1px 0 rgba(255,255,255,0.08)
   - 近阴影：0 2px 8px rgba(0,0,0,0.08)
   - 远阴影：0 8px 24px rgba(0,0,0,0.12)

## Task 3: 设计Token规范化

**目标**：统一圆角、间距、阴影等设计Token的使用，消除同页面内圆角层级过多的问题。

**文件**：
- `entry/src/main/ets/common/DesignTokens.ets`

**具体改动**：

1. **圆角层级精简**（当前有4/8/10/12/14/16/18/20/22/24等多种）：
   - XS=8（小元素：标签、徽章）
   - SM=12（卡片、消息气泡）
   - MD=16（容器、输入栏）
   - LG=20（大容器、面板）
   - FULL=999（圆形）
   - 修正ChatRadius.MESSAGE=18→12，ChatRadius.INPUT=22→20，与实际使用对齐

2. **字体系统完善**：
   - 添加缺失的MICRO=10fp
   - 统一消息正文行高为1.6（当前未显式设置）
   - AI回复文字色建议#D4D4E8（暗色）/ #212121（亮色），柔和于纯白

3. **间距系统微调**：
   - 当前Spacing系统合理，但组件中间距使用不统一
   - 统一卡片内padding为16vp，卡片间间距为12vp

## Task 5: 动画与微交互增强

**目标**：激活AnimationHelper中定义但未使用的动画参数，增强视觉动态感。

**文件**：
- `entry/src/main/ets/common/AnimationHelper.ets`
- `entry/src/main/ets/views/chat/ChatPage.ets`
- `entry/src/main/ets/components/chat/ChatMessageBubble.ets`
- `entry/src/main/ets/components/chat/ChatInputBar.ets`

**具体改动**：

1. **消息列表入场动画**：
   - 透明度0→1 + Y轴位移8vp→0
   - 300ms EaseOut，stagger间隔50ms

2. **Logo动画优化**（ChatPage）：
   - 当前：setInterval(50ms)手动计算，性能差
   - 改为：animateTo循环动画，使用repeat(-1)和alternate模式
   - 保留三重复合效果：缩放 + 浮动 + 旋转

3. **按钮按压反馈**：
   - 所有可点击按钮添加scale动画：按压时0.95，释放时1.0
   - 150ms EaseOut

4. **侧边栏功能入口stagger入场**：
   - 打开侧边栏时，功能项依次淡入
   - 每项延迟30ms，透明度0→1 + X轴位移-10vp→0

1. **Task 1（配色系统）** → 基础，其他任务依赖

## 预期效果

- 视觉统一性提升：消除两套颜色体系，所有颜色通过资源引用管理
- 液态玻璃质感升级：统一参数规范，增强深度层次和真实性
- 暗色模式可靠性：消除硬编码颜色导致的暗色模式适配问题
- 现代AI应用感：微动画、stagger入场、按压反馈等2026设计趋势
- 性能优化：Logo动画从setInterval改为animateTo

## 风险与注意事项

- 颜色资源引用修改需确保所有引用处同步更新，避免编译错误
- 液态玻璃参数调整需在真机上验证效果（模拟器效果可能不准确）
- 动画添加需注意性能影响，特别是消息列表stagger动画
- 旧架构清理需确认无其他地方引用
