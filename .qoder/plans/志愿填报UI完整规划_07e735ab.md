# 志愿填报板块 完整UI设计与功能修缮规划

## 现状分析

**已实现的6个页面：**
- `VolunteerPage.ets` — Hub主页（四宫格、成绩Banner、热门院校、时间线）
- `ScoreInputPage.ets` — 成绩录入（省份选择+三科输入）
- `VolunteerResultPage.ets` — 智能选志愿（模式Tab+筛选+分组列表+底栏）
- `VolunteerFormPage.ets` — 志愿表管理（列表+风险分析+底部操作）
- `ScoreLinePage.ets` — 分数线查询（搜索+年份/类型筛选+列表）
- `VolunteerDiagnosePage.ets` — 志愿诊断（环形评分+维度+建议）

**缺失的2个页面：**
- `MajorDetailPage.ets` — 专业详情页
- `CollegeVolunteerPage.ets` — 院校志愿详情页

**主要问题：**
1. 筛选逻辑未真正连接（VolunteerResultPage 的筛选不过滤数据）
2. 志愿表无拖拽排序功能（只有静态列表）
3. 诊断维度不支持展开/收起
4. 分数线搜索无防抖
5. 页面间数据传递未打通（如从结果页添加到志愿表）
6. 硬编码数据不够丰富，部分页面数据量太少

## 视觉与交互规范统一

所有新增/改造页面统一遵循：
- 液态玻璃卡片：`backgroundBlurStyle(BlurStyle.COMPONENT_REGULAR)` + `border({ width: 0.5, color: AppColors.GLASS_BORDER })`
- 按压反馈：`AnimConfig.LIQUID_PRESS_DURATION` / `LIQUID_PRESS_CURVE` / `PRESS_SCALE(0.96)`
- 三梯度色：冲刺 `#FF6B6B` / 稳妥 `#10B981` / 保底 `#5B7FFF`
- 渐变按钮：`linearGradient colors: [['#007AFF', 0.0], ['#5AC8FA', 1.0]]`
- 列表入场：Stagger动画 + `TransitionEffect.OPACITY.combine(TransitionEffect.translate({ y: 15 }))`
- 空状态：使用 EmptyView 组件或自建空状态（图标+文案+引导按钮）
