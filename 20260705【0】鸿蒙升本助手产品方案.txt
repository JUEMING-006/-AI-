鸿蒙升本AI助手 
一、核心设计理念
从了解到目标：从信息获取到最终录取，完整陪伴专升本全流程。
用户的升本旅程是：
先想清楚“为什么升”→了解“专升本是什么”→确认“我能考哪里”→规划“我怎么学”→执行“每天学什么”→最终“考后去哪”。
产品路径设计
二、完整用户决策路径 vs 功能映射
阶段	用户核心问题	功能模块	优先级
动机唤醒	我为什么要升本？值不值得？	信息呈现、政策解读、成功案例，失败案例	P0
认知建立	专升本是什么？考什么？怎么考？	动态信息聚合（政策/院校/科目）	P0
资格确认	我能考吗？跨地区行不行？	地区适配 + 资格智能过滤	P0
目标设定	我能报哪些院校和专业？	院校库 + 智能筛选匹配	P0
路径规划	我该怎么学？学什么？	AI动态学习规划 + 任务清单	P1
动态调整	我学得怎么样了？要调整吗？	任务完成度追踪 + 规划动态调整	P1
最终决策	我这个分数能上哪？	考后志愿匹配系统	P0

三、功能模块详细说明
模块一：信息获取与认知建立（P0）
解决用户的核心问题：“专升本是什么？我能考吗？值不值得考？”
加入一个院校的信息添加，可以获得该校的升本情况，与学校公布的信息。
功能点	说明
地区适配	用户选择所在省份后，系统自动匹配该省专升本政策、考试科目、报名时间、考试时间
资格智能检测	根据用户专业、年级、户籍等信息，自动判断是否符合报考条件，能否跨省报考
政策动态聚合	集中展示该省最新政策文件、报名通知、考试大纲，支持按时间/类别筛选
升本价值内容	展示专升本后的薪资提升数据、升学路径、成功案例，帮助用户建立动机
智能问答
（小艺）	用户可通过小艺语音提问：“专升本能跨省吗？”“XX专业考什么科目？”实时回复

模块二：院校与目标设定（P0）
解决用户的核心问题：“我能报哪些学校？我该选哪个作为目标？”
该学校对对于专升本的安排，或一些校内情况
功能点	说明
院校库	收录各省招收专升本的本科院校，包含：院校名称、办学性质（公办/民办/职业本科）、地址、招生电话、官网链接
专业对照	专科专业 → 可报考的本科专业对照表，支持用户输入专科专业自动匹配
智能筛选	按省份、办学性质、专业、城市类型等多维度筛选可报考院校
目标设定与收藏	用户可将心仪院校设为“目标院校”，系统记录目标列表
院校详情	展示历年录取分数线（近3年最低/平均/最高）、招生计划、考试科目、学费

模块三：AI动态学习规划（P1）
解决用户的核心问题：“我要考XX学校，现在开始该怎么学？”
功能点	说明
目标差距分析	基于目标院校历年录取线，自动计算当前水平与目标的分数差距
三阶段规划生成	根据考试时间自动生成“基础→强化→冲刺”三阶段学习计划，明确每阶段目标
每日任务清单	每日自动生成任务清单：今日学习内容、刷题数量、复习知识点、预计耗时
任务完成追踪	用户逐项勾选完成，系统记录完成情况（完成/部分完成/未完成）
动态调整机制	对比“每日预定清单 vs 实际完成清单”，未完成任务顺延至次日；每周汇总完成曲线，对下一周进行精细化安排
动态时间提醒	手机在线动态提醒，根据用户设定的学习时间推送通知，记录各时段完成情况
数据可视化	展示本周任务完成率趋势图，让用户直观看到学习节奏和调整轨迹

模块四：考后志愿匹配（P0）
解决用户的核心问题：“我考了这个分数，能上哪些学校？”
功能点	说明
成绩录入	用户输入各科目真实考试成绩，系统自动计算总分
智能三梯度匹配	基于历年录取数据，自动推荐：冲刺院校、稳妥院校、保底院校
录取概率预测	每个推荐院校显示预测录取概率（高/中/低）和概率百分比
筛选与排序	按省份、办学性质、专业、录取概率等多维度筛选排序
院校详情对比	对比多个院校的录取线、学费、招生计划等关键信息
志愿方案生成	用户勾选心仪院校，生成完整志愿方案，支持一键导出
碰一碰分享	同学之间碰一碰互传志愿方案，交流讨论

四、鸿蒙特性集成
鸿蒙特性	应用场景	优先级
小艺智能体	语音唤醒：“专升本能跨省吗？”“给我推荐广东的院校”	P0
互动卡片	桌面卡片展示：倒计时、今日任务、最新政策提醒	P0

实况窗	锁屏显示：今日任务完成度、倒计时	P1
智能填充	信息填写时智能补全（省份/院校/专业名称）	P2


五、数据来源策略
核心原则：人工录入核心数据 + AI辅助校验 + 用户反馈修正。
数据类型	来源	更新频率	维护方式
院校基本信息	各省教育考试院/院校官网	每年	人工录入
录取分数线（近3年）	各省教育考试院	每年	人工录入 + AI异常值校验
招生计划	各省教育考试院	每年	人工录入
专业对照表	各省教育考试院	每年	人工录入
政策文件	各省教育考试院	实时	爬虫监控 + 人工审核
考试大纲	各省教育考试院	不定期	人工录入

AI在数据中的角色：当用户查询“XX院校录取线”时，AI从已录入的数据库中调取（准确率100%），而非联网搜索。AI联网搜索仅作为辅助验证机制，用于检测已录入数据是否存疑（如某校分数线波动异常时报警提示人工复核）。
数据覆盖范围（MVP阶段）：3个目标省份 × 每省20所核心院校 × 近3年录取数据。

八、参赛核心卖点
维度	差异化优势
用户路径	严格遵循专升本用户真实决策链，非功能堆砌
AI创新	动态规划调度 + 可视化调整轨迹，让评委“看见”AI在持续工作
鸿蒙特性	小艺智能体 + 桌面卡片 + 应用接续 + 碰一碰
数据壁垒	人工精校 + AI校验 + 用户反馈修正，保证数据准确性
垂直深耕	专为专升本群体定制，非高考工具简单移植
