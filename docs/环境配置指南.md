## 快速开始

### 1. 克隆项目后首次配置

```bash
# 1. 进入项目目录
cd TopUpAI

# 2. 创建本地配置文件（复制示例文件）
cp entry/src/main/ets/common/ApiConfig.example.ets entry/src/main/ets/common/ApiConfig.local.ets
cp entry/src/main/ets/common/ApiConfig.example.ets entry/src/main/ets/common/ApiConfig.prod.ets

# 3. 修改本地配置（根据实际情况）
# 编辑 ApiConfig.local.ets，配置你的本地后端地址
```

### 2. 配置本地开发环境

编辑 `entry/src/main/ets/common/ApiConfig.local.ets`：

```typescript
export class ApiConfigEnv {
  // 你的本地后端地址
  static readonly BASE_URL: string = 'http://localhost:8000';
  
  // API 路径前缀
  static readonly API_PREFIX: string = '/api/v1';
  
  // 请求超时时间
  static readonly TIMEOUT: number = 10000;
  
  // 启用调试模式
  static readonly DEBUG_MODE: boolean = true;
  
  // 后端未完成时使用 Mock 数据
  static readonly USE_MOCK_DATA: boolean = true;
  
  // API 密钥（本地开发通常不需要）
  static readonly API_KEY: string = '';
}
```

### 3. 启动项目

```bash
# 在 DevEco Studio 中运行项目
# 项目会自动加载 ApiConfig.local.ets 配置
```

## 配置文件说明

### 📁 文件结构

```
entry/src/main/ets/common/
├── ApiConfig.ets           # ⭐ 主配置文件（自动加载环境配置）
├── ApiConfig.example.ets   # 📝 配置示例（提交到 Git，供参考）
├── ApiConfig.local.ets     # 🔒 本地开发配置（不提交，包含本地配置）
└── ApiConfig.prod.ets      # 🔒 生产环境配置（不提交，包含生产配置）
```

### 🔐 Git 忽略规则

`.gitignore` 已配置以下文件不提交：

```gitignore
# API 配置文件（包含敏感信息）
**/ApiConfig.local.ets
**/ApiConfig.prod.ets
```

### 📊 配置项详解

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `BASE_URL` | string | `http://localhost:8000` | API 服务器基础地址 |
| `API_PREFIX` | string | `/api/v1` | API 路径前缀 |
| `TIMEOUT` | number | `10000` | 请求超时时间（毫秒） |
| `DEBUG_MODE` | boolean | `true` | 是否启用调试模式（打印日志） |
| `USE_MOCK_DATA` | boolean | `true` | 是否使用 Mock 数据（后端未完成时） |
| `API_KEY` | string | `''` | API 密钥（如果后端需要） |

## 环境切换

### 🔧 开发环境（默认）

在 `ApiConfig.ets` 中：

```typescript
import { ApiConfigEnv } from './ApiConfig.local';
```

**特点：**
- 连接本地后端 `http://localhost:8000`
- 启用调试模式
- 可使用 Mock 数据

### 🚀 生产环境

在 `ApiConfig.ets` 中修改导入：

```typescript
import { ApiConfigEnv } from './ApiConfig.prod';
```

**特点：**
- 连接生产环境 API
- 关闭调试模式
- 使用真实后端数据

### 🔄 切换流程图

```
开发阶段 → 联调测试 → 生产部署
   ↓           ↓           ↓
 local      local       prod
 Mock=true  Mock=false  Mock=false
 Debug=true Debug=true  Debug=false
```

## API 接入指南

### 📝 在代码中使用配置

```typescript
import { ApiConfig } from '../common/ApiConfig';

// 获取配置
console.log('API 地址:', ApiConfig.BASE_URL);
console.log('完整路径:', ApiConfig.getFullUrl('/colleges'));

// 检查是否使用 Mock 数据
if (ApiConfig.USE_MOCK_DATA) {
  console.log('当前使用 Mock 数据');
} else {
  console.log('当前使用真实 API');
}

// 调试模式检查
if (ApiConfig.DEBUG_MODE) {
  console.log('调试信息:', someData);
}
```

### 🔌 Service 层改造示例

**改造前（纯 Mock）：**

```typescript
export class CollegeService {
  getColleges(): Promise<CollegeInfo[]> {
    // 直接返回 Mock 数据
    return Promise.resolve(MOCK_COLLEGES);
  }
}
```

**改造后（支持 Mock/API 切换）：**

```typescript
import { ApiConfig } from '../common/ApiConfig';
import { httpClient } from './HttpClient';

export class CollegeService {
  async getColleges(): Promise<CollegeInfo[]> {
    // 如果配置为使用 Mock 数据，返回模拟数据
    if (ApiConfig.USE_MOCK_DATA) {
      return Promise.resolve(MOCK_COLLEGES);
    }
    
    // 否则调用真实 API
    try {
      const response = await httpClient.get('/colleges');
      return response.data as CollegeInfo[];
    } catch (error) {
      console.error('获取院校列表失败:', error);
      throw error;
    }
  }
}
```

### 🎯 完整的 Service 改造模板

```typescript
import { ApiConfig } from '../common/ApiConfig';
import { httpClient } from './HttpClient';
import { hilog } from '@kit.PerformanceAnalysisKit';

const DOMAIN = 0x0001;
const TAG = 'YourService';

export class YourService {
  /**
   * 获取数据（支持 Mock/API 切换）
   */
  async getData(params: any): Promise<any> {
    // 打印调试信息
    if (ApiConfig.DEBUG_MODE) {
      hilog.info(DOMAIN, TAG, 'getData called with params: %{public}s', JSON.stringify(params));
    }
    
    // Mock 数据模式
    if (ApiConfig.USE_MOCK_DATA) {
      if (ApiConfig.DEBUG_MODE) {
        hilog.info(DOMAIN, TAG, 'Using Mock data');
      }
      return this.getMockData(params);
    }
    
    // 真实 API 调用
    try {
      const response = await httpClient.get('/your-endpoint', { params });
      
      if (ApiConfig.DEBUG_MODE) {
        hilog.info(DOMAIN, TAG, 'API response: %{public}s', JSON.stringify(response.data));
      }
      
      return response.data;
    } catch (error) {
      hilog.error(DOMAIN, TAG, 'API call failed: %{public}s', error.message);
      throw error;
    }
  }
  
  /**
   * Mock 数据方法
   */
  private getMockData(params: any): any {
    // 返回模拟数据
    return { /* mock data */ };
  }
}
```

## 常见问题

### ❓ Q1: 首次克隆项目后找不到配置文件？

**A:** 需要手动创建配置文件：

```bash
cp ApiConfig.example.ets ApiConfig.local.ets
cp ApiConfig.example.ets ApiConfig.prod.ets
```

### ❓ Q2: 如何确认当前使用的是哪个环境？

**A:** 查看应用启动时的日志输出：

```
=== API Configuration ===
BASE_URL: http://localhost:8000
API_PREFIX: /api/v1
DEBUG_MODE: true
USE_MOCK_DATA: true
========================
```

### ❓ Q3: 为什么我的配置没有生效？

**A:** 检查以下几点：

1. 确认 `ApiConfig.ets` 中导入的是正确的配置文件
2. 确认配置文件名称正确（`ApiConfig.local.ets` 不是 `ApiConfig.local.ets.txt`）
3. 重启应用使配置生效

### ❓ Q4: 如何添加新的配置项？

**A:** 按以下步骤操作：

1. 在 `ApiConfigEnv` 类中添加新配置项
2. 在 `ApiConfig` 类中导出新配置项
3. 更新 `ApiConfig.example.ets` 示例文件
4. 更新 `ApiConfig.local.ets` 和 `ApiConfig.prod.ets`

### ❓ Q5: 生产环境部署需要注意什么？

**A:** 部署前检查：

- [ ] 修改 `ApiConfig.ets` 导入 `ApiConfig.prod`
- [ ] 配置真实的生产环境 API 地址
- [ ] 设置 `USE_MOCK_DATA = false`
- [ ] 设置 `DEBUG_MODE = false`
- [ ] 配置正确的 API 密钥
- [ ] 测试所有接口是否正常

### ❓ Q6: 如何在团队中共享配置？

**A:** 
- 提交 `ApiConfig.example.ets` 到 Git（作为模板）
- 团队成员各自创建自己的 `ApiConfig.local.ets`
- 在文档中说明配置项的含义和示例值
- 敏感信息（密钥、真实地址）不要提交到 Git
