## 1. 设计系统

### 1.1 色彩体系

整体采用 **Apple 风格单色系**，以品牌蓝 `#007AFF` 为核心色，辅以中性灰构建干净克制的视觉基底。v2.0 在此基础上扩展了渐变背景色系和彩色液态玻璃变体，为页面营造更具沉浸感的视觉层次。

#### 品牌色

| 名称 | 色值 | 用途 |
|------|------|------|
| PRIMARY_COLOR | `#007AFF` | Apple 蓝，主按钮、选中态、强调文字 |
| PRIMARY_LIGHT | `#E6F2FF` | 浅蓝背景，Tab 选中态背景 |
| PRIMARY_GRADIENT_END | `#5AC8FA` | 渐变终止色，按钮/卡片渐变 |

#### 背景与表面色

| 名称 | 色值 | 用途 |
|------|------|------|
| BG_HOME_TOP | `#EEF2FF` | 首页顶部渐变起始色（极浅蓝灰） |
| BG_HOME_BOTTOM | `#FFFFFF` | 首页底部渐变终止色（纯白） |
| BG_TAB | `#F0F4FF` | Tab 内容页背景（浅蓝灰，较旧版更亮） |
| BG_SUB | `#F5F5F7` | 子页面背景（浅灰） |

### 1.6 液态玻璃效果（Liquid Glass）

这是整个应用的**核心视觉语言**。v2.0 将液态玻璃从卡片扩展到更多元素类型，实现全界面沉浸式玻璃质感。

#### 玻璃变体与适用场景

| 变体 | 色值 | 不透明度 | 模糊度 | 适用场景 |
|------|------|----------|--------|----------|
| XLIGHT | `#F5FFFFFF` | 96% | COMPONENT_THICK | 标题栏、导航栏、顶部区域 |
| LIGHT | `#E8FFFFFF` | 90% | COMPONENT_REGULAR | 标准内容卡片 |
| MEDIUM | `#CCFFFFFF` | 80% | COMPONENT_REGULAR | 悬浮面板、浮层 |
| DARK | `#99F5F5F7` | 60% | COMPONENT_REGULAR | 底部抽屉、遮罩卡片 |
| XDARK | `#66F5F5F7` | 40% | COMPONENT_THICK | 强力模糊遮罩 |
| PRIMARY | `#18FFFFFF` | 9% | COMPONENT_REGULAR | 品牌色玻璃叠加（头部区域微染） |
| SUCCESS | `#18F0FDF4` | 9% | COMPONENT_REGULAR | 绿色玻璃叠加（保底相关） |
| WARNING | `#18FFF8E1` | 9% | COMPONENT_REGULAR | 橙色玻璃叠加（提醒相关） |
| RUSH | `#18FFF5F5` | 9% | COMPONENT_REGULAR | 红色玻璃叠加（冲刺相关） |
| STABLE | `#18F0FDF4` | 9% | COMPONENT_REGULAR | 绿色玻璃叠加（稳妥相关） |
| SAFE | `#18EEF2FF` | 9% | COMPONENT_REGULAR | 蓝色玻璃叠加（保底相关） |

#### 玻璃通用视觉层

所有玻璃元素统一遵循以下视觉层叠规范：

1. **背景色**：根据场景选择对应玻璃变体的色值
2. **背景模糊**：根据变体选择 COMPONENT_THIN(20) / COMPONENT_REGULAR(40) / COMPONENT_THICK(80)
3. **边框**：0.5px 宽，颜色 `#33FFFFFF`（微白描边）
4. **圆角**：根据元素大小选择对应圆角
5. **阴影**：根据悬浮层级选择 CARD / ELEVATED / GLASS
6. **内边距**：通常 20vp（大卡片）或 12vp（小卡片/列表项）
7. **可选高光条**：顶部 1px 渐变高光，从 `#4DFFFFFF` 到透明，增强立体感

#### 玻璃高光条

在玻璃元素顶部叠加一条极细高光线，模拟真实玻璃的反光质感：

```
┌──────────────────────────────┐  ← 顶部 1px 高光线（#4DFFFFFF → transparent）
│                              │
│    玻璃卡片内容区域            │
│    背景色：#E8FFFFFF           │
│    模糊度：COMPONENT_REGULAR   │
│    边框：0.5px #33FFFFFF       │
│                              │
└──────────────────────────────┘
```

高光条实现方式：在卡片内容顶部叠加一个 1px 高度的渐变层，方向从上到下，颜色从 `#4DFFFFFF` 渐隐至透明。

#### 彩色玻璃叠加应用

彩色玻璃变体主要用于以下场景的**微染色**效果：

- **进度条/环形图背景**：使用对应梯度色的玻璃变体作为轨道色
- **标签徽章**：使用对应功能色的玻璃变体作为背景
- **选中态指示**：使用 PRIMARY 玻璃变体高亮选中项
- **头部区域微染**：Banner 卡片背景叠加极淡的品牌蓝玻璃

#### 玻璃边框系统

| 场景 | 边框宽度 | 边框颜色 |
|------|----------|----------|
| 标准卡片 | 0.5px | `#33FFFFFF` |
| 选中卡片 | 1.5px | 对应功能色（如 PRIMARY_COLOR） |
| 输入框聚焦 | 1.5px | PRIMARY_COLOR |
| 按钮 | 0px | 无边框 |
| 悬浮 Tab 栏 | 1px | `#33FFFFFF` |

### 1.7 动画系统

动画是本应用交互体验的核心。所有可点击元素都有按压反馈，列表有逐项入场动画，数字有弹性增长效果。

#### 液态按压效果（卡片/列表项）

| 状态 | 缩放 | 透明度 | 时长 | 曲线 |
|------|------|--------|------|------|
| 按下 (Down) | 0.91 | 0.75 | 80ms | 弹性缓出 |
| 释放 (Up) | 1.0 | 1.0 | 400ms | 弹性缓出 |

> 按下时缩放到 0.91，产生明显的"下沉感"；释放时 400ms 弹性缓出，产生"弹性回弹"。

#### 按钮按压效果

| 状态 | 缩放 | 透明度 | 时长 | 曲线 |
|------|------|--------|------|------|
| 按下 (Down) | 0.88 | 0.70 | 60ms | 弹性缓出 |
| 释放 (Up) | 1.0 | 1.0 | 500ms | 弹性缓出 |

> 按钮按压更"深"（0.88）、更快响应（60ms）、回弹更慢更弹性（500ms）。

#### Tab 栏按压效果

| 状态 | 缩放 | 透明度 | 时长 | 曲线 |
|------|------|--------|------|------|
| 按下 (Down) | 0.85 | 0.60 | 60ms | 弹性缓出 |
| 释放 (Up) | 1.0 | 1.0 | 450ms | 弹性缓出 |

#### 弹性数字动画

| 属性 | 值 |
|------|----|
| 时长 | 600ms |
| 曲线 | Spring 弹性 |
| 适用 | 倒计时数字、总分、评分数字 |

> 数字变化时使用 Spring 弹性曲线，产生轻微过冲再回弹的效果，增强活力感。

#### 进度条生长动画

| 属性 | 值 |

## 2. 应用架构与导航

### 2.1 整体架构

应用采用**单 Navigation + 悬浮 Tab 栏**架构：

```
┌─────────────────────────────┐
│                         ┌───┐│
│                         │ T ││
│     页面内容（全屏）      │ a ││
│                         │ b ││
│                         └───┘│  ← 悬浮 Tab 栏（底部居中）
```

- 底层：全屏页面内容（4 个 Tab 页面切换）
- 顶层：悬浮 Tab 栏（固定在底部居中位置，覆盖在内容之上）
- Tab 栏容器设置穿透交互，使空白区域不拦截点击

### 2.2 悬浮 Tab 栏设计

#### 布局

- 位置：底部居中，距底部 20vp + 安全区高度
- 宽度：4 个 Tab 项宽度 + 3 个间距 + 2 × 20vp 内边距
- 高度：64vp
- 层级：最高层级

## 5. 交互模式汇总

### 5.1 按压反馈模式

所有可点击元素统一采用以下按压反馈模式：

1. **卡片/列表项（液态级）**：0.91 缩放 + 0.75 透明度，80ms 按下 / 400ms 释放
2. **按钮（按钮级）**：0.88 缩放 + 0.70 透明度，60ms 按下 / 500ms 释放
3. **Tab 项（Tab 级）**：0.85 缩放 + 0.60 透明度，60ms 按下 / 450ms 释放
4. **收藏/快捷入口（微弹跳级）**：1.3 倍放大 → 回弹，150ms 两段式动画

### 5.2 列表入场动画模式

- 逐项延迟 = 索引 × 50ms
- 效果：透明度 0→1 + Y 轴 20vp→0
- 适用：任务列表、推荐卡片列表

### 5.3 数字增长动画模式

- 从 0 到目标值递增
- 每帧 +1，间隔 20ms
- 曲线：Spring 弹性（v2.0 新增）
- 适用：诊断评分

### 5.4 展开收起动画模式

- 透明度 + Y 轴 10vp 位移
- 过渡时长：约 300ms
- 曲线：缓入缓出
- 适用：分数线专业展开、诊断维度展开、推荐卡片展开

### 5.5 进度条增长动画模式

- 从 0 到目标百分比
- 500ms 弹性缓出
- 逐项延迟入场
- 适用：学习阶段环形进度、柱状图

### 5.6 页面转场模式

- 内容区透明度淡入 + 微缩放（0.97→1.0）+ 微位移（Y 15vp→0）
- 过渡时长：300ms
- 曲线：快速出慢速入
- 适用：Tab 页面切换

## 9. 面向用户群体的设计考量

### 9.1 目标用户画像

- **人群**：专科在校生，准备专升本考试
- **年龄**：18-22 岁
- **特征**：手机高频使用者、对 UI 美感有期待、备考压力大需激励
- **核心需求**：清晰的信息层级、简洁的操作路径、温暖的鼓励感

### 9.2 设计语言适配

| 考量维度 | 设计决策 |
|----------|----------|
| 色彩温度 | 保留 Apple 蓝冷色调为主，但增加暖色光斑和渐变过渡，避免过于冰冷 |
| 信息密度 | 保持当前适度留白，卡片间距统一，避免信息过载引发焦虑 |
| 正向激励 | 倒计时、进度条、环形图使用蓝色（冷静）+ 绿色（达成），少用红色警告 |
| 操作路径 | 4 个 Tab 入口清晰，快捷入口 8 宫格一目了然，减少跳转层级 |
| 反馈即时性 | 所有可点击元素 80ms 内响应按压，让用户感到"跟手" |
| 视觉舒适度 | 液态玻璃营造轻盈通透感，减轻备考沉重氛围 |

### 9.3 色彩情感映射

## 10. 动效设计原则

### 10.1 动效层级

| 层级 | 用途 | 典型时长 | 曲线 |
|------|------|----------|------|
| 细节级 | 按钮按压、标签切换、chip 选中 | 60-80ms | 弹性缓出 |
| 组件级 | 卡片展开/收起、弹窗进出 | 300ms | 缓入缓出 |
| 页面级 | Tab 切换、页面推入弹出 | 250-300ms | 快速出慢速入 |
| 数据级 | 数字变化、进度条生长 | 500-600ms | Spring 弹性 |

### 10.2 动效设计原则

1. **即时响应**：所有触摸反馈在 80ms 内开始动画，让用户感到"跟手"
2. **弹性自然**：释放动画使用 400-500ms 的弹性缓出，模拟物理回弹
3. **层次分明**：不同元素使用不同时长和曲线，避免千篇一律
4. **克制使用**：入场动画仅在首次加载时触发，不干扰日常滚动
5. **性能优先**：骨架屏 shimmer 注意循环停止，列表动画限制最大延迟在 500ms 内

## 11. 液态玻璃高级效果

### 11.1 玻璃高光条

在卡片顶部叠加 1px 渐变高光，模拟真实玻璃的反光：

```
┌──────────────────────────────┐ ← 高光条（1px，#4DFFFFFF → transparent）
│                              │
│    玻璃卡片内容区域            │
│                              │
└──────────────────────────────┘
```

适用场景：渐变按钮、大型 Banner 卡片、头部信息卡片。

### 11.2 玻璃外发光

输入框聚焦、选中卡片等场景使用外发光效果：

- 边框：1.5px 主色
- 外发光：4px 扩散，主色半透明
- 动画：200ms 渐显

### 11.3 彩色玻璃微染

在标准白色玻璃基础上叠加极淡的彩色玻璃层（9% 不透明度），为特定场景增添色彩氛围：

- 进度条轨道：使用对应功能色的玻璃变体
- 选中态指示：使用 PRIMARY 玻璃变体
- 头部区域：使用 PRIMARY 玻璃变体微染
- 梯度标签：使用对应梯度色的玻璃变体

### 11.4 动态光斑（首页）

首页顶部区域使用 2-3 个径向渐变圆形，配合高斯模糊，营造柔和的光影氛围：

- 光斑 1：蓝色，半径 80vp，位置偏左上
- 光斑 2：紫色，半径 60vp，位置偏右上
- 光斑 3（可选）：绿色，半径 40vp，位置居中偏下
- 模糊度：40vp
- 交互：可随设备陀螺仪轻微偏移（如支持）

## 12. 性能与体验平衡

### 12.1 毛玻璃性能策略

| 场景 | 模糊度选择 | 原因 |
|------|------------|------|
| 大量列表卡片 | COMPONENT_REGULAR (40) | 平衡效果与性能 |
| 标题栏/导航栏 | COMPONENT_THICK (80) | 仅一个元素，可接受强模糊 |
| 滚动列表中的卡片 | 优先使用 GLASS_XLIGHT | 接近纯白，模糊计算量小 |
| Tab 栏 | COMPONENT_THICK (80) | 仅一个元素 |

### 12.2 动画性能策略

| 注意事项 | 做法 |
|----------|------|
| Stagger 延迟上限 | 限制在 500ms 以内（10 项 × 50ms） |
| 动画循环 | 骨架屏 shimmer 仅在加载态使用，加载完成后停止 |
| 滚动 + 动画 | 列表项入场动画仅在首次加载时触发 |
| 页面切换 | 使用透明度而非位移动画，减少重绘 |
